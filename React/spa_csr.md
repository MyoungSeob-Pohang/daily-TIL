# SPA & CSR

보통 기존의 웹사이트은 서버에게 요청을 하면 요청받은 페이지의 html을 가져다 주는 형식의 MPA : Multi Page application 이였다.  
이 말은 우리가 서버에게 주소로 /home 을 주면 서버가 home.html을 가져다 주고 /about로 요청하면 about.html을 반환준다는 것이다.  
하지만 리액트는 SPA : Single Page Application로 한개의 페이지만 가지고 있다. 그래서 클라이언트가 요구사항을 말해도 한가지의 페이지만 줄 수있다.  
만약 페이지를 이동하고 싶어서 새로운 페이지를 요청해도 서버가 한페이지밖에 안들고 있어서 하나만 던져준다. 그럼 SPA는 페이지이동을 할 수없을까 ?  
이동 매커니즘은 간단하다.

1. 어떤 요청을 하던 똑같은 index.html 을 던져준다.
2. 그런데 던져줄때 REACT APP이라는 것을 통째로 던져준다.
3. 만약 posts라는 페이지로 이동하고 싶어서 어떤 버튼을 클릭하였다.
4. REACT APP이 index.html을 통쨰로 업데이트 시켜버린다.

그러니까 페이지이동이 원래 서버와 통신하여 가져오는 것인데 리액트는 브라우져가 혼자 처리해버린다. 그래서 페이지 이동을 할때 서버와 통신을 하고 그것을 기다릴 필요가 없기 때문에 페이지 전환자체가 굉장히 빠르고 핸드폰에서 사용하는 것과 같이 새로고침이 이루어지지 않는다. 만약 서버와 주고받아햘 데이터가 있다면 일단 페이지 먼저 바꿔버리고 그 후에 데이터만 수신받아서 업데이트 시킨다. 그래서 속도가 굉장히 빠르다. 이렇게 클라이언트가 직접 페이지를 업데이트 시켜버리는 방식을 CSR : Client Side Rendering 이라고 한다.  
즉 리액트는 SPA 이면서 CSR방식을 사용하는 것이다.
